# Agent Swarm Template

A production-ready template for orchestrating multiple AI coding agents working in parallel using git worktrees, [Beads (bd)](https://github.com/steveyegge/beads) for persistent memory and task tracking, tmux for session management, and Claude Code for execution.

Clone this repo, initialize it in your project, and start shipping features 10x faster with a self-organizing team of AI agents.

---

## How It Works

```
You → /swarm "Build a SaaS app with auth, billing, dashboard, and API"
                          │
                 Orchestrator Agent
           ┌──────────┼──────────┐
           ▼          ▼          ▼
     agent-auth   agent-api  agent-dash
     (worktree)   (worktree) (worktree)
           │          │          │
           └──────────┼──────────┘
                      ▼
                main branch ← merges land here
```

The orchestrator:
1. Breaks your task into subtasks with a dependency graph (using Beads)
2. Creates isolated git worktrees for each unblocked task
3. Opens a tmux session with one pane per agent
4. Dispatches Claude Code agents into each pane
5. Monitors progress, runs standups
6. Merges completed work, promotes newly unblocked tasks
7. Repeats until done

You review standup reports, approve merges, and steer direction. The agents do the grinding.

## Quick Start

### 1. Clone and initialize

```bash
git clone https://github.com/yourorg/agent-swarm-template my-app
cd my-app
./scripts/swarm-init.sh
```

### 2. Plan and dispatch agents

```bash
claude
> /swarm Build a full-stack task management app with auth, REST API, React frontend, and PostgreSQL backend
```

The orchestrator will:
- Create an epic with subtasks in Beads
- Set up the dependency graph
- Spawn agents for all unblocked tasks

### 3. Monitor your swarm

```bash
# Watch agents work in real time
tmux attach -t my-app-swarm

# Quick status check from Claude
> /progress

# Full standup report
> /standup
```

### 4. Merge completed work

```bash
> /merge
```

### 5. Repeat until done

The cycle continues: standup → merge → dispatch → standup → ... until all tasks are closed.

### 6. Clean up

```bash
> /teardown
```

## Commands

| Command | Description |
|---------|-------------|
| `/swarm <task>` | Plan and dispatch agents for a task |
| `/standup` | Full progress report across all agents |
| `/progress` | Quick status check (data only) |
| `/spawn <task-id>` | Spawn a single agent on a specific task |
| `/merge` | Merge completed agent branches into main |
| `/teardown` | Remove all worktrees and kill all agent sessions |

## Configuration

After running `swarm-init.sh`, edit `.swarm-config.json` to customize:

```json
{
  "project_name": "my-app",
  "project_dir": "/path/to/my-app",
  "swarm_dir": "/path/to/my-app-swarm",
  "tmux_session": "my-app-swarm",
  "max_agents": 4,
  "created": "2026-03-01T12:00:00Z"
}
```

- **max_agents**: Maximum concurrent agents (default: 4). Start small, scale up.
- **swarm_dir**: Parent directory for git worktrees (sibling to your project).
- **tmux_session**: Name of the tmux session for agent panes.

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| **Beads over markdown plans** | Structured dependency graph, hash-based IDs, `bd ready` in milliseconds |
| **Git worktrees over clones** | Shared `.git` dir, faster creation, less disk space |
| **tmux over background processes** | Visual observability, ability to intervene, scroll output |
| **Max 4 agents** | Practical limit for merge complexity and human review overhead |

## GitHub Actions

Two workflows are included:

- **Daily Standup** (`daily-standup.yml`): Runs at 9am UTC on weekdays, generates a standup report and posts it as a GitHub Issue.
- **Agent CI** (`agent-ci.yml`): Runs tests on every push to `agent-*` branches. Auto-detects project type (Node, Python, Go, Rust).
